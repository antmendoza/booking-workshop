# TODO comments must be actionable

In exercise code, TODO comments must only mark
actionable items — code the learner needs to
write. Informational or contextual notes must
use "Note:" (or plain comments) instead of TODO.

**Why:** TODOs that describe already-written code
or provide background confuse learners about what
they actually need to implement. Every TODO should
correspond to a real delta between exercise and
solution.

**How to apply:** When writing or reviewing
exercise files, verify that every TODO maps to
code the learner must add. If a comment only
provides context, explains how surrounding code
works, or reminds the learner of a dependency,
use a plain comment or "Note:" prefix instead.
